## Data

Per-individual JSON metrics from `ml_out_var/metrics/test_*.json`, parquet predictions from `ml_out_var/predictions/`.

Each held-out individual has two alleles scored (maternal + paternal) with personal variants inserted. Both alleles share the same target SSU values since SSU is measured at the junction level, not allele-specific. Sites with SSU=777 (sentinel) are excluded.

## Tissue Specificity

This analysis quantifies tissue specificity for each splice site and examines how prediction accuracy varies with tissue specificity. Two complementary approaches are used:

- **n_tissues_detected** — how many of the 5 evaluation tissues a site appears in. No thresholds, no imputation. Includes all 50,010 sites.
- **tau** (Yanai et al. 2005) — a continuous tissue specificity metric computed only over the tissues where a site is detected. Stratified by n_tissues_detected to keep tau values comparable within each stratum.

Both approaches avoid the problem of assigning SSU=0 to tissues where a site wasn't detected — absence could mean the gene wasn't expressed (unknown SSU) rather than the site being unused (SSU=0).

### Data preparation

Starting from `df_all` (all site × sample × tissue predictions), we:
1. Remove sentinel SSU=777 entries. The value 777.0 is a sentinel that marks splice sites where no reliable SSU can be computed. It arises in two ways during the pipeline:
   - **Low read coverage**: in `build_matrix.py`, when the denominator (total junction-spanning reads) at a splice site falls below the coverage threshold (`denom_thresh`), the SSU ratio would be unreliable or undefined, so it is set to 777.0 instead of computing inc/total. This means the site was observed in the RNA-seq data but with too few reads to estimate usage.
   - **GENCODE-filled sites**: in `fill_gencode_sites.py`, the pipeline ensures every annotated GENCODE splice site appears in the training data, even if it was never observed in a particular sample's RNA-seq. Sites that are annotated in GENCODE but absent from a sample's splicing matrix are inserted with SSU=777.0. This means the site is known to exist in the reference annotation but was not detected in that individual.

   In both cases, 777.0 tells the model "there is no ground truth here" — the training loss masks these entries, and evaluation metrics exclude them. Note: `splice_sites.parquet` files already exclude 777 entries (filtered by `extract_splice_sites.py`), so this filter is a safety check.
2. Deduplicate alleles (maternal/paternal) per site × tissue × sample, keeping one observation
3. Compute per-site × per-tissue statistics: `n_detected` (individuals with valid SSU, out of ~10), `mean_ssu`, `median_ssu`, `sd_ssu`, `min_ssu`, `max_ssu`
4. Count how many tissues each site has at least one valid individual in (`n_tissues_detected`, 1–5)

This yields ~50,010 unique splice sites on test chromosomes. Saved to `data/results/site_tissue_summary.csv`.

#### S7. Tissue specificity groups

Sites detected in all 5 tissues (13,600) are classified into 3 groups based on tau:

- **Constitutive** (tau < 0.1): similar SSU across all tissues
- **Intermediate** (0.1 ≤ tau < 0.5): moderate tissue variation
- **Tissue-specific** (tau ≥ 0.5): strong tissue-differential splicing

These groups connect the data overview (Figure 2 in paper) to the model benchmark (Figure 3 in paper) — the same grouping is used for both the SSU distribution panel and the stratified model metrics.

![Tau SSU violin 5-tissue](data/figures/main/tau_ssu_violin_5tissue.png)

**Figure.** SSU value distributions stratified by tau bin (8 equal-count quantile bins across 13,600 five-tissue sites).
